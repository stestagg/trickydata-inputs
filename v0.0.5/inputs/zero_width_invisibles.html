zero_width_invisibles

This is a run of five code points that each render with zero advance width: ZERO WIDTH SPACE (U+200B), ZERO WIDTH NON-JOINER (U+200C), ZERO WIDTH JOINER (U+200D), WORD JOINER (U+2060), and ZERO WIDTH NO-BREAK SPACE / BOM (U+FEFF). The decoded text is several code points long but paints nothing, so any layout or width calculation keyed off byte or code-point count over-reports its size, while cursor positioning and selection can land on positions that are visually indistinguishable. Several are Default_Ignorable, so conforming renderers must not show a "missing glyph" box for them. Unicode lists the Default_Ignorable_Code_Point property in the Unicode Character Database: https://www.unicode.org/reports/tr44/#Default_Ignorable_Code_Point
