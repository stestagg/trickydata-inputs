real_world_url_ipv4_octal

The leading zero makes the first IPv4 part octal under the WHATWG parser, so `0300.0.2.1` becomes the TEST-NET address `192.0.2.1`. Other libraries have historically treated leading-zero parts as decimal or rejected them, making this a security-relevant parser differential for host allowlists. The WHATWG IPv4 algorithm specifies octal prefixes. CPython's CVE-2021-29921 tracked the security consequences of accepting ambiguous leading-zero IPv4 text in `ipaddress`: https://url.spec.whatwg.org/#concept-ipv4-parser https://github.com/python/cpython/issues/80565
