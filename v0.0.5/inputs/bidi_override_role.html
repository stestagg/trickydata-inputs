bidi_override_role

This string places RIGHT-TO-LEFT OVERRIDE (U+202E) before the logical ASCII sequence "resu" and POP DIRECTIONAL FORMATTING (U+202C) after it. Many renderers therefore display the bounded sequence as "user", making the input appear similar to its paired ASCII baseline despite different stored bytes. Unicode Standard Annex #9 defines explicit directional overrides: https://www.unicode.org/reports/tr9/ Unicode Technical Report #36 discusses bidi visual spoofing: https://www.unicode.org/reports/tr36/tr36-15.html#Bidirectional_Text_Spoofing
