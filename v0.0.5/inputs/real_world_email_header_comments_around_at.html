real_world_email_header_comments_around_at

This is a valid RFC 5322 message-header addr-spec: dot-atoms permit CFWS, including comments, on either side of the at-sign. It is not RFC 5321 SMTP envelope mailbox syntax. Treating all "email address" contexts as identical either rejects a valid header or leaks comments into an envelope address. RFC 5322 sections 3.2.3 and 3.4.1 define dot-atom CFWS and addr-spec: https://www.rfc-editor.org/rfc/rfc5322.html#section-3.2.3 https://www.rfc-editor.org/rfc/rfc5322.html#section-3.4.1
