real_world_email_maximum_local_part

The local-part is exactly 64 ASCII octets, the maximum an SMTP implementation must accept. Off-by-one checks, character-count assumptions, and database columns sized for common addresses can reject or truncate it. RFC 5321 section 4.5.3.1.1 sets the local-part limit to 64 octets: https://www.rfc-editor.org/rfc/rfc5321.html#section-4.5.3.1.1
