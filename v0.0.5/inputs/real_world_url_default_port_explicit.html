real_world_url_default_port_explicit

Port 443 is the default for HTTPS, so this spelling is equivalent to the paired URL with no explicit port. Parsers that retain one spelling but not the other can create duplicate cache or policy entries. RFC 3986 section 6.2.3 covers scheme-based default-port normalization: https://www.rfc-editor.org/rfc/rfc3986.html#section-6.2.3
