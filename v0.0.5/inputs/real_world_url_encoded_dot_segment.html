real_world_url_encoded_dot_segment

The WHATWG special-URL parser treats `%2e%2e` as a double-dot segment and serializes this URL as `/a/b/marker`. A generic RFC 3986 normalizer does not percent-decode a path before dot-segment removal, so parser choice changes the normalized path. The WHATWG URL Standard explicitly recognizes percent-encoded dot segments: https://url.spec.whatwg.org/#double-dot-path-segment
