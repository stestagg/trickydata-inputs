invalid_utf8_chunk_two_continuations

Two continuation bytes cannot form a UTF-8 string by themselves. Appending this chunk to the paired F4 90 prefix produces F4 90 80 80, the invalid encoding of U+110000. It exercises stream validators that classify chunks independently or lose upper-bound state between buffers. See the Unicode well-formed UTF-8 byte sequences in Table 3-7: https://www.unicode.org/versions/Unicode15.0.0/ch03.pdf
