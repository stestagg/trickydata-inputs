real_world_invalid_email_empty_unquoted_local_part

The local-part before the at-sign is empty and therefore cannot match an SMTP Dot-string or Quoted-string. The paired `""` local-part is valid, which catches validators that discard quote syntax before checking emptiness. RFC 5321 section 4.1.2 requires a non-empty Dot-string but permits an empty Quoted-string: https://www.rfc-editor.org/rfc/rfc5321.html#section-4.1.2
