real_world_url_query_crlf_injection

The query contains percent-encoded CRLF followed by an inert marker header. A URL library must not turn decoded query data into HTTP header structure. The `.invalid` host prevents an accidental request from reaching a service. CPython's CVE-2019-9740 allowed CRLF injection through attacker-controlled URL query text passed to `urllib`: https://github.com/python/cpython/issues/80457
