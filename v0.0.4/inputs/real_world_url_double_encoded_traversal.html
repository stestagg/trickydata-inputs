real_world_url_double_encoded_traversal

One percent-decoding pass turns `%252e%252e%252f` into `%2e%2e%2f`; a second turns it into `../`. Pipelines in which a proxy, router, and application decode at different stages can therefore expose traversal that no individual layer sees in the original text. RFC 3986 section 2.4 warns that a percent-encoded string must not be decoded more than once: https://www.rfc-editor.org/rfc/rfc3986.html#section-2.4
