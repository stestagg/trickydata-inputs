invalid_utf8_obsolete_five_byte_sequence

This is shaped like the five-byte UTF-8 forms allowed by early historical definitions. Modern UTF-8 is limited to one through four bytes and Unicode scalar values no greater than U+10FFFF, so the 0xF8 starter is invalid. This catches decoders that still implement the obsolete 31-bit form. See RFC 3629, which restricts UTF-8 to four bytes: https://www.rfc-editor.org/rfc/rfc3629
